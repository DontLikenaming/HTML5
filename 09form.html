폼 관련 태그

개요

html에서 사용자의 정보를 입력받아 서버로 전송하려면 form 요소를 사용해야 함

form 태그를 사용해서
입력받은 데이터는 서버로 어떻게 보낼 것인지(method)
전송한 데이터의 유형은 무엇인지(enctype)
전송된 데이터는 서버에서 누가 처리할지(action) 등을 지정해야 함

입력받은 데이터 서버로 보내기

  • get 방식은 url에 질의문자열(querystring) 형태로
    데이터를 작성해서 전송함.
    따라서, 입력한 데이터가 그대로 노출됨.
    또한, 전송할 수 있는 데이터의 크기가 제한적임.
    중요도가 낮은 정보를 입력 폼 없이 임의로 전달 시 주로 사용
  • post 방식은 데이터를 http header에 별도로 작성해서 전송함.
    따라서, 데이터가 외부로 보이지 않으며, 전송할 수 있는 데이터의 크기가 무제한.
    또한, 텍스트와 바이너리를 같이 보낼 수 있어서 보안성과 활용성이 좋음

질의문자열

클라이언트가 서버로 정보를 요청 시
정보전달 양식에 의해 작성된 문자열을 의미.
질의문자열의 시작은 ?로, 전달하려는 데이터는 키와 값의 형태로 작성하며
각각의 정보는 &로 연결함. 영문자, 숫자, 일부 특수문자는 그대로 전송되지만,
그외 나머지는 %와 16진수코드로 인코딩된 문자로 변환되어 전송됨
(예: 수지는 %EC%88%98%EC%A7%80로 인코딩됨 - urlencoder.org)

전송 데이터 유형

데이터를 전송할 때 enctype 속성으로 전송 데이터의 유형을 지정할 수 있음
텍스트 데이터 전송 시 enctype은 생략 가능
텍스트와 바이너리 데이터 전송 시 multipart/form-data로 설정


데이터 처리 주체(handler) 지정

action 속성에는 전송된 데이터들을 서버의 어떤 프로그램이 맡아 처리할 것인지 지정


폼 요소 그룹화 태그

fieldset 태그는 폼을 구성하는 다양한 폼 요소들을 목적에 따라 그룹화하는 데 사용됨

또한 legend 태그로 그룹화한 폼들에 대해 이름을 지정할 수 있음

다양한 input 태그

html5에서는 다양한 input 요소 태그를 사용해서 사용자로부터 데이터를 입력받을 수 있음

  • text : 한줄 텍스트를 입력받음
  • password : 한줄 비밀 텍스트를 입력받음 입력시 별표문자가 화면에 표시
  • radio : 여러 선택사항 중 하나만 입력받음
  • checkbox : 여러 선택사항 중 다수를 입력받음
  • select : 여러 선택사항 중 하나만 입력받음 단, 선택할 수 있는 사항은 드롭다운 메뉴로 보여짐
  • textarea : 여러 줄 텍스트를 입력받음
  • button : 사용자가 클릭할 수 있는 버튼을 표시하며 type 속성으로 버튼의 종류를 지정함.
    button은 일반적인 버튼을 표시, 보통 자바스크립트와 함께 사용
    submit은 입력한 데이터들을 서버로 전송시 사용
    reset은 입력한 데이터들을 모두 지우고 초기화할 때 사용
  • file : 텍스트가 아닌 파일을 서버로 전송할때 사용
    단, enctype이 multipart/form-data로 설정되어야 함
    또한, 서버측에서는 텍스트만 처리하는 방식이 아닌
    다른 방식으로 처리해야 함
  • label : 폼 요소에 제목을 작성할때 사용

공통 input 속성

value : 폼 요소의 기본값을 지정

readonly : 폼 요소에 입력되어 있는 값을 수정 못하게 막아놓음

disabled : 해당 폼 요소 자체를 사용할 수 없음(뭐가 입력되어 있어도 전송되지 않음)

placeholder : 폼 요소 내에 미리 설정해둔 기본값이 표시되게 함

size, maxlength : 폼 요소의 너비와 입력 가능한 최대 글자수를 지정

required : 폼 요소에 값을 반드시 입력해야 하는지의 여부를 지정
